## 做什么

把"想法"变成"文档"，这是建站最重要的一步。

## 什么是 PRD？

**PRD（Product Requirements Document，产品需求文档）** 是你跟 AI 之间的"设计蓝图"。没有 PRD，AI 只能猜你想要什么；有了 PRD，AI 能精准地一次到位。

一个好的 PRD 包含：
- **这是什么网站**（一句话说清）
- **给谁用**（目标用户）
- **有什么功能**（功能清单）
- **长什么样**（风格参考）
- **先做什么、后做什么**（优先级）

## 提示词模板

> 🎯 **推荐 Skill**：先安装 `create-prd`，执行 `npx skills create-prd` 按提示填写即可自动生成 PRD，效果比直接提问更好。

```bash
# 安装 Skill（一键复制）
npx skills add phuryn/pm-skills@create-prd -g -y
```

安装后执行以下命令启动 PRD 向导：

```bash
npx skills create-prd
```

如果不想用 Skill，也可以直接把以下模板复制给 AI：

请帮我写一份产品需求文档（PRD），项目是 [网站名称]。

我的背景：我是一个完全不会编程的小白，想用 AI 帮我建站。

网站信息：
- 网站类型：[个人博客 / 作品集 / 电商 / 公司官网]
- 主要用途：[描述核心目的]
- 目标用户：[谁会来看/用这个网站]
- 我希望包含的功能：
  1. [功能 A]
  2. [功能 B]
  3. [功能 C]

风格偏好：[简洁 / 科技感 / 温馨 / 复古]

请按标准的 PRD 结构输出：摘要 → 背景 → 目标 → 功能需求 → 非功能需求 → 发布计划。

## 📋 更多提示词模板

**提示词 1：定义目标用户画像**

```text
我是一个完全不会编程的小白，想用 AI 帮我建站。

请帮我分析并定义我的网站的目标用户。

我的网站信息：
- 网站类型：[个人博客 / 摄影作品集 / 电商 / 企业官网]
- 核心功能：[一句话说明网站做什么]
- 我希望吸引什么样的人：[描述你心目中的理想访客]

请帮我：
1. 定义 1-2 个典型用户画像（给用户起个名字，描述他的年龄、职业、上网习惯）
2. 分析这些人访问我网站时最想找到什么
3. 根据用户画像，建议网站应该突出什么内容
4. 建议用什么语气和风格跟这些用户沟通

用表格呈现用户画像，让我能直接用在 PRD 中。
```

**提示词 2：功能优先级排序**

```text
我是一个完全不会编程的小白，想用 AI 帮我建站。

我计划做一个 [网站类型]，我列出了以下想要的功能，请帮我按优先级排序。

我的功能清单：
1. [功能 A]
2. [功能 B]
3. [功能 C]
4. [功能 D]
5. [功能 E]
6. [功能 F]

我的目标是在 [X] 周内上线第一个版本。

请帮我：
1. 按 MVP（最小可行产品）原则，把这些功能分成 P0（必须有）、P1（应该有）、P2（可以有）三档
2. 每个功能给出为什么放在这个档次的理由
3. 建议第一版应该包含哪些功能
4. 后续版本可以逐步添加哪些功能

请用表格呈现。

**任务**：选一个你想做的网站（个人博客 / 作品集 / 小店展示页），使用本节的提示词模板，让 AI 帮你生成一份 PRD。然后你自己阅读并回答：
- 这份 PRD 跟你想的一样吗？
- 有没有漏掉的功能？
- 优先级排序你同意吗？
