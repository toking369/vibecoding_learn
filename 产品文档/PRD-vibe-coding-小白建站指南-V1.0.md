# PRD: 《小白用 Vibe Coding 建站 — 从零到上线完全指南》

---

## 1. 摘要

本产品是一份面向零编程基础人群的实操指南，教他们用 **Vibe Coding（AI 辅助编程）** 的方式从零开始搭建并发布自己的网站。文档定位为"手把手教程"，覆盖从工具安装、需求梳理、AI 对话生成代码，到最终部署上线的完整路径。目标是让完全不懂技术的人也能在几小时内拥有自己的网站。

## 3. 背景

### 3.1 这是什么？

2024–2025 年，AI 编程工具（如 Claude、Cursor、GitHub Copilot、Windsurf 等）的能力突飞猛进。**Vibe Coding** 这个词由 AI 教父 Andrej Karpathy 提出，指的是一种全新的编程方式：**你不再需要手写每一行代码，而是用自然语言描述你想要什么，AI 帮你生成代码。**

对于想建网站但不会编程的人来说，这意味着一场革命。

### 3.2 为什么要现在做？

过去，建一个网站需要：

1. 学习 HTML / CSS / JavaScript（至少几个月）
2. 理解服务器、域名、部署等概念（数周）
3. 反复调试、修复 bug（数天到数周）

而现在，有了 AI 编程助手，**一个完全不懂代码的人，在有人引导的情况下，完全可以在几小时内完成从想法到上线**。

现状是：

- 网上教程大多仍假设读者有编程基础
- 很多 "AI 编程教程" 讲得太技术化，小白看不懂
- 缺少一份**完整的、从零开始的、实操导向**的指南

### 3.3 这件事为什么可能？

近半年 AI 代码生成质量大幅提升：

- Claude 的 Sonnet / Opus 系列能生成完整的 HTML/CSS/JS 页面
- Cursor / Windsurf 等 IDE 内置 AI，可以实时对话修改代码
- 部署工具（Vercel / Netlify / GitHub Pages）让发布变成点几下鼠标

**门槛从未如此之低。**

#### 第一章：Vibe Coding 到底是什么？

**目标：消除陌生感，建立信心**

- **什么是 Vibe Coding？**
  - 一句话定义：像跟朋友聊天一样告诉 AI 你想要什么网站，它帮你写代码
  - 核心不是你写代码，而是**你描述、AI 生成、你验收**
- **故事引入**：一个完全不会编程的人，用 2 小时做出自己的博客的真人真事
- **为什么现在可以？**（简单版）
  - AI 变聪明了
  - 工具变简单了
  - 部署变免费了
- **常见的误区澄清**
  - ❌ "这是作弊" → ✅ "这是新技能"
  - ❌ "生成的代码质量不行" → ✅ "满足个人项目完全够用"
  - ❌ "还是需要学编程" → ✅ "理解基本概念有帮助，但不是必须"

#### 第二章：建站前的准备工作（手把手）

**目标：完成所有工具安装和账号注册**

##### 2.1 你需要的东西（清单）

| 项目        | 推荐                          | 费用     | 难度 |
| ----------- | ----------------------------- | -------- | ---- |
| 电脑        | 任何 Windows / Mac 电脑       | 已有     | ★☆☆  |
| 浏览器      | Chrome（推荐）或 Edge         | 免费     | ★☆☆  |
| AI 编程工具 | **Claude Code** 或 **Cursor** | 有免费版 | ★★☆  |
| 代码编辑器  | VS Code（附赠）               | 免费     | ★★☆  |
| 版本控制    | GitHub 账号                   | 免费     | ★★☆  |
| 部署平台    | Vercel 或 Netlify 账号        | 免费     | ★★☆  |

##### 2.2 分步安装教程（带截图指引）

1. **注册 Claude / Cursor 账号**
   - 打开官网 → 点击注册 → 用 Google / GitHub / 邮箱注册
   - 安装桌面版（如果有）
2. **安装 VS Code**
   - 官网下载 → 双击安装 → 默认选项下一步
3. **注册 GitHub**
   - 用户名建议用心：`yourname` 就行
4. **注册 Vercel**
   - 用 GitHub 账号直接登录（最简单）

> **特别提醒**：每一步都告诉用户"如果卡住了怎么办"——用截图、用搜索、用 AI 本身求助。

#### 第三章：从想法到网站的完整流程

**目标：完成第一个可用网站**

##### 3.1 第一步：想清楚你要什么

在跟 AI 对话之前，先想清楚这三个问题：

| 问题                 | 例子                                |
| -------------------- | ----------------------------------- |
| 这个网站是干什么的？ | "我的个人博客" / "我的摄影作品展示" |
| 谁来看这个网站？     | "我的朋友和家人" / "潜在客户"       |
| 最重要的功能是什么？ | "能展示照片" / "有联系方式"         |

✏️ **模板**：读者可以填写的"需求卡片"

##### 3.2 第二步：第一次跟 AI 对话

**黄金提问模板：**

```
我想建一个 [网站类型]，它的目标是 [用途]。

具体要求：
1. 它应该包含 [功能 A]
2. 它应该能 [功能 B]
3. 我希望它的风格是 [描述风格，如简洁/活泼/专业]

请帮我生成完整的 HTML 文件，包含样式。
```

📌 **实际案例对比**：

- ❌ "帮我做个网站"（太模糊）
- ✅ 上面这种"填空式"提问（AI 能准确理解）

##### 3.3 第三步：查看和预览

- 如何保存 AI 生成的代码
- 如何用浏览器打开 HTML 文件预览
- 如何用 VS Code 打开项目文件夹

##### 3.4 第四步：提出修改意见

- 修改的"话术"模板：

```
这个页面挺好的，但我想做几个调整：
1. 把标题颜色改成蓝色
2. 在导航栏加一个"关于我"的链接
3. 页脚加上版权声明

请直接输出修改后的完整代码。
```

- **原则**：一次不要提超过 3-4 个修改，分批进行更稳定

##### 3.5 第五步：反复迭代直到满意

- 循环：预览 → 提修改意见 → AI 改 → 再预览
- 大约 3-5 轮迭代就能得到一个不错的页面

##### 3.6 第六步：部署上线（激动人心的时刻）

- 在 Vercel 上点几下鼠标就能上线
- **流程**：
  1. 把文件夹拖到 Vercel
  2. 等待几秒钟
  3. 得到一个网址（例如 `my-site.vercel.app`）
  4. 发给朋友看看！

#### 第五章：常见问题与避坑指南

**目标：让读者遇到问题时能自己解决 90% 的情况**

##### 常见问题 Q&A

| 问题                        | 快速解答                                  |
| --------------------------- | ----------------------------------------- |
| AI 生成的代码打开是空白的？ | 检查是否保存为 `.html` 文件，用浏览器打开 |
| 图片显示不出来？            | 图片路径要用相对路径或在线链接            |
| 样式跟我想要的不一样？      | 给 AI 更具体的描述（颜色值、参考网站）    |
| 页面在手机上不好看？        | 告诉 AI "加响应式设计，适配手机"          |
| 代码太多看不懂？            | 不需要看懂，只要能用就行                  |
| AI 把我的代码改坏了？       | 保留之前的版本，重新告诉 AI 你的需求      |

##### 十大避坑清单

1. ✅ **先小后大**：从一个页面开始，不要一上来就想做淘宝
2. ✅ **经常保存**：每次改完把代码保存好
3. ✅ **一次一个需求**：不要一次性提 10 个修改
4. ✅ **学会"回退"**：改坏了就回到上一个版本
5. ✅ **用浏览器检查**：右键 → 检查 → 看 Console（控制台）有没有红字
6. ✅ **善用截图**：把问题截图发给 AI 让它帮你诊断
7. ✅ **注意隐私**：不要在生产网站里放密码、密钥
8. ✅ **备份大法**：每次重大改动前复制一份文件夹
9. ✅ **保持简单**：能用一页解决的不要做五页
10. ✅ **享受过程**：这是创造，不是考试
